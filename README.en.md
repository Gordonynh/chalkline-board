# Chalkline Board

Chalkline Board is a local-first classroom whiteboard for teaching, annotation, blank writing, and image explanation. It is split from the ClearBoard/OpenWhiteboard code line as an independent public project and does not include textbook images, school-internal resources, or third-party branded assets.

## Features

- Blank board and local image import
- Pen, highlighter, eraser, laser pointer, and pan mode
- Undo, page management, note saving, and image/PDF export
- Unified mouse, touch, and stylus input
- Windows desktop shell with portable builds and standard Inno Setup installers

## Development

```bash
npm ci
npm run dev
npm run build
npm run release:installer
```

Installers are generated by `scripts/release.mjs` through Inno Setup 6 `ISCC.exe`. The project does not use a custom C# installer.

## Local Classroom Desktop Deployment

For the local classroom desktop setup on this machine, use:

```bash
npm run deploy:desktop
npm run verify:desktop
```

Deployment contract:

- `deploy:desktop`, `deploy:classroom`, and `desktop:deploy` all point to the same guarded classroom deployer; do not use the lower-level `desktop:publish*` scripts as a classroom release substitute.
- The textbook app must be published as a single-file EXE on the desktop named `步步高v<version>.exe`.
- The administrator desktop must contain only the latest `步步高v*.exe`; stale textbook EXEs are removed during deployment.
- The public desktop must not contain any `步步高v*.exe`; it only receives the classroom shortcuts.
- The whiteboard desktop shortcut must be `鸿合白板软件.lnk` and point to the same-version blank whiteboard multifile app.
- The visualizer desktop shortcut must be `鸿合视频展台.lnk` and point to the same-version visualizer multifile app.
- Both the administrator desktop and the public desktop shortcuts must be updated.
- Duplicate classroom shortcuts with the same managed prefixes are removed during deployment and rejected by verification.
- `deploy:desktop` runs the preflight suite before publishing and then runs the strict desktop verifier after updating the desktop.
- `desktop:contract` is part of preflight and checks the script-level deployment contract before any desktop release.
- `verify:desktop` checks version alignment, stale desktop EXE cleanup, textbook resources, visualizer bundle markers, exact shortcut targets, and the SHA256 match between the desktop EXE and the latest single-file textbook artifact.
- To verify a specific release, pass the version through npm: `npm run verify:desktop -- --version 0.613.19`.
